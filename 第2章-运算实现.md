计算机组成原理-第2章运算实现

数据的表示与运算

一、数制表示与转换

- 16进制：前缀 0x 或尾部加 H

- 十进制：末尾加 D（可省略）

- 二进制：末尾加B

- 十进制转二进制：整数部分「除基取余法」，小数部分「乘基取整法」

- 技巧：先转二进制，再转八进制/十六进制，效率更高

二、定点数与浮点数

1. 定点数

小数点位置固定，分为无符号数和有符号数，核心特点：只有无符号整数，不存在无符号小数。

2. 浮点数

小数点位置可浮动，本质是二进制科学计数法，用于表示范围更大的数值。

三、有符号定点数的编码（原码、反码、补码、移码）

有符号定点数由符号位+数值位组成，最高位为符号位（0正1负），小数点隐含在指定位置（末尾或最前），四种编码规则如下：

- 正数：补码 = 原码 = 反码

- 负数：补码 = 反码 + 1（反码为原码除符号位外按位取反）

- 移码：补码的符号位按位取反（0变1，1变0），便于对比数的大小

四、各种编码的作用

- 补码：解决减法运算复杂的问题，结合加法器实现减法（减法转等价加法）

- 移码：用于快速对比数的大小（无需考虑符号位，直接比较数值即可）

- C语言关联：定点整数int 按补码方式存储；int 转 unsigned 时，存储内容不变，仅改变解释方式（补码→无符号定点数）

五、C语言中定点数的相关操作

- a: 0x000286a1（真值165537），c: 0x86a1（真值-31071）；b: 0xfff7751（真值-34991），d: 0x7751（真值30545）

六、数据长度扩展（零扩展与符号扩展）

- 无符号整数：扩展时高位直接填0（零扩展）

- 有符号整数：扩展时高位填充与符号位一致的数字（符号扩展，适用于补码）

示例：带符号整数（补码）90（0,1011010）→ 符号扩展后为0,000000001011010；补码-90（1,0100110）→ 符号扩展后为1,11111110100110。

二、多路选择器（MUX）

核心功能：从多个输入信号中，通过控制信号选择一个信号输出，常见类型有2输入1输出、4输入1输出等。

示例：4输入1输出MUX，通过2位控制信号（00、01、10、11），选择4路输入中的1路输出；8位输入的MUX同理，通过3位控制信号选通。

三、三态门与非门的区别

核心区别：非门没有控制信号，只有输入和输出；三态门有控制信号，可通过控制信号实现“非运算”（输出端加小圆圈），还可实现高阻态。

加法器的实现

一、加法器分类

- 串行进位的并行加法器：n个一位全加器串联组成，支持n位并行加法；缺点是进位信息串行产生，运算速度慢。

- 并行进位的并行加法器：在串行进位基础上优化，增加CLA（先行进位）部件，进位信息并行产生，运算速度更快。

- 带标志位的加法器：在并行进位加法器基础上，增加逻辑电路，输出OF、SF、ZF、CF等标志位，用于判断运算结果。

二、一位全加器（FA）基础

核心逻辑：实现1位二进制加法，输入为被加数A、加数B、低位进位Cin，输出为和S、高位进位Cout。

- 和：S = A⊕B⊕Cin

- 进位：Cout = A·B + (A⊕B)·Cin

三、带标志位的加法器（核心考点）

四个核心标志位含义（必考）：

- OF（Overflow Flag，溢出标志）：判断带符号数加减运算是否溢出，OF=1表示溢出，OF=0表示未溢出。

- SF（Sign Flag，符号标志）：判断带符号数加减运算结果的正负，SF=1表示结果为负，SF=0表示结果为正。

- ZF（Zero Flag，零标志）：判断加减运算结果是否为0，ZF=1表示结果为0，ZF=0表示结果不为0。

- CF（Carry Flag，进位/借位标志）：判断无符号数加减运算是否溢出，CF=1表示溢出（有进位/借位），CF=0表示未溢出。

四、加法器实现总结

核心：加法器是运算器的基础，串行进位速度慢，并行进位（CLA）速度快；带标志位加法器可通过标志位判断运算结果的有效性。

逻辑算数单元ALU

一、ALU核心功能与实现原理

- 功能：支持算术运算（加、减、乘、除等）和逻辑运算，共支持k种功能。

- 实现原理：多个功能电路 + 多路选择器（MUX），通过控制信号选通MUX的某条线路，实现对应功能。

- 核心参数：
        

  - 输入：两个n位运算数A、B（与机器字长相同）

  - 控制信号：由控制器通过指令译码产生，位数m ≥ [log₂k]（k为功能数量）

  - 输出：n位运算结果（与机器字长相同）+ 辅助信息（如高位进位）

二、考点补充

- CPU主要由运算器和控制器组成，ALU是运算器的核心。

- 加法器是ALU的核心，加减乘除运算均基于加法实现（减法转加法，乘法转移位+加法，除法转移位+减法）。

- PSW（程序状态字）标志寄存器：存储ALU运算产生的标志位（OF、SF、ZF、CF等），供控制器判断后续操作。

三、定点数补码加减运算规则（核心）

- 同号相加：数值部分（绝对值）相加，符号位不变；若结果符号与加数符号相反，说明溢出。

- 异号相加：数值部分用绝对值大的减绝对值小的，符号位与绝对值大的数一致。

- 减法转加法：将减数取补（负补），转变为被加数 + 减数的补码。

- 溢出判断（三种方法）：
        

  - 方法1：两个正数相加结果为负，或两个负数相加结果为正，说明溢出。

  - 方法2：符号位进位Cs与最高数值位进位C1不同（Cs≠C1），说明溢出。

  - 方法3：双符号位补码，运算结果两个符号位不同（01或10），说明溢出；相同（00或11），说明未溢出。

加减法的运算器电路

核心逻辑：通过多路选择器（MUX）控制加数B的输入，实现加法和减法的切换。

- 加法（Sub=0）：直接将被加数A和加数B输入加法器，按位相加，输出结果F和进位Cout。

- 减法（Sub=1）：将减数B按位取反，末位加1（得到[-B]补），再与被加数A相加，实现减法运算。

示例：4bit补码，X=3（0011），Y=4（0100）；X+Y=0111（7D），X-Y=0011 + 1011（[-4]补）= 1111（-1D）。

乘法的实现

一、计算机实现乘法的三种方式

- 移位运算 + 加减运算：通过多次移位和加法，等效实现乘法（速度较慢）。

- 阵列乘法器：快速乘法器的一种，由多个全加器组成阵列，可在1个时钟周期内完成乘法运算（速度快）。

- 其他快速乘法器：原理类似阵列乘法器，均可在1个时钟周期内完成运算。

二、阵列乘法器详解

核心特点：以4×4位阵列乘法器为例，被乘数X（4bit）和乘数Y（4bit）输入后，通过阵列中的全加器并行运算，直接输出8bit乘积P。

扩展：32×32位阵列乘法器原理相同，仅全加器数量和阵列规模增加，可快速完成32bit数的乘法运算。

三、规格化浮点数

- 核心要求：阶码不能全为0或全为1，尾数隐含最高位为1（规格化表示，节省1位存储）。

- 范围：分为正上溢、负上溢、正下溢、负下溢，溢出判断以阶码溢出为准（尾数溢出可通过规格化纠正）。

浮点数的加减运算

一、运算步骤（核心）

1. 对阶：小阶向大阶靠齐（保证尾数运算的正确性），阶码差为ΔE，小阶尾数右移ΔE位，阶码变为大阶。

2. 尾数加减：按原码加减规则，对对齐后的尾数进行加减运算。

3. 尾数规格化：若尾数溢出或未规格化，通过左规（尾数左移，阶码减1）或右规（尾数右移，阶码加1）调整。

4. 尾数舍入：对多余的尾数位进行舍入处理，IEEE 754默认舍入模式为“就近舍入”（类似0舍1入）。

5. 溢出判断：以阶码是否溢出为准，右规后阶码全1为上溢，左规后阶码全0为下溢。

二、舍入模式（IEEE 754默认）

就近舍入：类似“0舍1入”，若舍弃的3位刚好为100，则看尾数末位：末位为0则直接截断，末位为1则截断后末位加1。

三、运算示例

已知float X=2⁻¹²⁶ + 2⁻¹⁴⁹，Y=-2⁻¹²⁶（符合IEEE 754标准），计算Z=X+Y的过程：

1. 对阶：X阶码与Y阶码相等（ΔE=0），无需对阶。

2. 尾数加减：X尾数为1.00000000000000000000001（隐含1），Y尾数为-1.00000000000000000000000，相加后尾数为+0.00000000000000000000001。

3. 尾数规格化：尾数需左规，但阶码当前为00000001，左规后阶码变为00000000（非规格化），因此保持阶码为00000000，尾数不变。

4. 尾数舍入：无需舍入，保持尾数原样。

5. 溢出判断：阶码为00000000（非规格化），无溢出，最终Z为非规格化正数。

四、关键结论

浮点数的溢出不以尾数溢出来判断，尾数溢出可通过规格化操作纠正；运算结果是否溢出，核心看阶码是否上溢（右规后阶码全1）或下溢（左规后阶码全0）。
