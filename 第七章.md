2. I/O 接口与内部寄存器
2.1 I/O 接口（控制器）
I/O 接口是连接 CPU 与外设的部件，通常是一块芯片，存在 I/O 控制器标准。
2.2 内部寄存器
数据寄存器：缓冲数据
控制寄存器：接收 CPU 命令
状态寄存器：返回设备状态（就绪 / 忙 / 错误）
3. 三种 I/O 控制方式
3.1 程序查询方式
CPU 不断轮询设备状态，直到准备好再传输。
流程CPU 执行 → 启动 I/O → 循环查询 → 数据传输 → 结束
特点
CPU 忙等
效率极低
只适合低速设备（键盘）
3.2 程序中断方式
设备准备好后发送中断，CPU 再处理。
流程CPU 执行 → 启动 I/O → I/O 准备 → 指令周期末尾发中断 → CPU 处理中断
特点
CPU 可并行工作
效率中等
适合中速设备（打印机）
3.3 DMA 方式
主存与外设之间建立直接数据通路，不经过 CPU。
流程CPU 预处理 → DMA 传输 → 完成后发一次中断 → CPU 后处理
特点
几乎不占用 CPU
速度快
适合高速设备（磁盘）

6. I/O 端口编址方式
6.1 统一编址（存储器映射）
I/O 端口占用主存地址空间
使用普通访存指令访问
优点：指令通用
缺点：占用内存地址
6.2 独立编址（I/O 映射）
I/O 地址与内存地址完全独立
使用专用 IN/OUT 指令
优点：不占内存地址
缺点：需专用指令
