## 2. I/O接口与内部寄存器
### 2.1 I/O接口的作用
I/O接口（控制器）是连接CPU与外设的桥梁，通常是一块专用芯片，包含标准协议。

### 2.2 内部寄存器结构
+-------------------+| I/O 接口芯片 |+-------------------+| 数据寄存器 (DR) | <-- 缓存 CPU 与外设之间的数据| 控制寄存器 (CR) | <-- 接收 CPU 的控制命令| 状态寄存器 (SR) | <-- 返回设备状态（就绪 / 忙 / 错误）+-------------------+

## 3. 三种I/O控制方式
### 3.1 方式对比表
| 控制方式 | CPU状态 | 适用设备 | 效率 |
| :--- | :--- | :--- | :--- |
| 程序查询 | 全程忙等 | 低速（键盘） | 低 |
| 程序中断 | 并行工作 | 中速（打印机） | 中 |
| DMA | 仅前后处理 | 高速（磁盘） | 高 |

### 3.2 程序查询方式流程
开始↓设置地址、计数器↓启动外设↓读取设备状态↓设备就绪？→ 否 → 循环查询↓ 是传送数据↓修改地址、计数器↓传送完成？→ 否 → 回到读取状态↓ 是结束
plaintext

### 3.3 程序中断方式流程
主程序执行↓启动外设（设备准备中，CPU 继续执行）↓设备准备完成 → 发送中断请求↓CPU 在指令周期末尾响应中断↓进入中断服务程序（处理数据）↓中断返回，继续执行主程序
plaintext

## 4. DMA直接内存访问
### 4.1 DMA核心原理
DMA（Direct Memory Access，直接内存访问）是一种让**主存与高速外设直接传输数据，不经过CPU**的控制方式。

### 4.2 DMA系统架构
+---------+ +---------+| CPU | | 主存 |+----+----+ +----+----+| |v v+---------------------------+| 主存总线 |+-----------+---------------+|v+-----------------+| DMA 控制器（接口）|+--------+--------+|v+-----------------+| I/O 总线 |+--------+--------+|v+-----------------+| 高速磁盘 |+-----------------+
plaintext

### 4.3 DMA工作流程
【1. 预处理】CPU 设置：
主存地址
设备地址
数据长度
启动外设和 DMA 控制器
【2. 数据传输】DMA 控制器接管总线主存与外设之间直接传输数据块CPU 继续执行其他程序
【3. 后处理】传输完成，DMA 控制器向 CPU 发送中断CPU 执行中断服务程序，处理后续工作
plaintext

## 5. 中断系统
### 5.1 中断的基本流程
中断请求：外设向 CPU 发送请求信号
中断判优：CPU 判断优先级，决定响应哪一个请求
中断响应：CPU 在指令周期末尾响应，执行中断隐指令
关中断
保存断点（PC）
转入中断服务程序入口
中断处理：执行中断服务程序，完成数据处理
中断返回：恢复现场，开中断，回到主程序
plaintext

### 5.2 中断优先级排序
优先级从高到低：
硬件故障中断
非屏蔽中断
DMA 请求
可屏蔽中断（I/O 设备）
软件中断（系统调用）
plaintext

## 6. I/O端口编址方式
### 6.1 统一编址（存储器映射）
内存地址空间结构：+-------------------+ 高地址| I/O 端口区域 |+-------------------+| 内存单元区域 |+-------------------+ 低地址
plaintext
- 特点：I/O端口占用部分主存地址空间，使用普通访存指令访问
- 优点：指令通用，无需专用I/O指令
- 缺点：占用主存地址，译码复杂

### 6.2 独立编址（I/O映射）
内存地址空间：独立的地址范围I/O 地址空间：独立的地址范围
plaintext
- 特点：I/O端口与主存地址完全独立，使用IN/OUT专用指令访问
- 优点：不占用主存地址，译码速度快
- 缺点：需要专用I/O指令支持
