       |  I/O 总线   |<-->|  I/O 接口   |<-->|  低速外设   |
       +------+------+    +-------------+    +-------------+
                           +-------------+
                           |  高速外设   |
                           +-------------+
DMA：Direct Memory Access 直接内存访问
DMA 接口 = DMA 控制器，是一种特殊的 I/O 控制器
主存 ↔ 高速 I/O 设备之间有直接数据通路（DMA 总线）
CPU 向 DMA 发读写命令，指明主存地址、设备地址、数据量
DMA 自动控制传输，完成一整块才发一次中断
三种 IO 控制方式对比
plaintext
【程序查询方式】
CPU 执行 → 启动 I/O → CPU 查询等待 → 传输 I/O 数据
→ I/O 准备与传送全程串行

【程序中断方式】
CPU 执行 → 启动 I/O → I/O 准备 → CPU 继续执行
→ 指令周期结束 → 中断请求 → CPU 处理中断

【DMA 方式】
CPU 执行 → 启动 I/O → I/O 准备
→ 存取周期结束 → DMA 请求 → 直接实现 I/O ↔ 主存

plaintext

优点：指令通用、灵活

plaintext

优点：不占内存、译码快
缺点：需专门 I/O 指令

程序中断方式
中断：CPU 执行程序时，暂停当前程序，处理紧急请求，处理完返回继续执行。
流程：
中断请求
中断响应（判优）
中断处理
中断返回
中断分类
plaintext
广义中断
├── 内中断（异常/陷入）→ CPU 内部
│   ├── 自愿中断（指令中断，如系统调用）
│   └── 强迫中断（硬件故障、软件中断，如除0、缺页）
│
└── 外中断（狭义中断）→ CPU 外部
    ├── 外设请求（I/O 完成）
    └── 人工干预
可屏蔽中断：
IF=1 开中断（允许）
IF=0 关中断（不允许）
中断优先级
硬件故障中断最高
软件中断次之
非屏蔽中断 > 可屏蔽中断
DMA > 普通 I/O
高速设备 > 低速设备
输入设备 > 输出设备
实时设备 > 普通设备

单重中断与多重中断
plaintext
【单重中断】
关中断 → 保存断点 → 送向量 → 保护现场
→ 执行服务 → 恢复现场 → 开中断 → 返回

【多重中断】
关中断 → 保存断点 → 送向量 → 保护现场+屏蔽字
→ 开中断 → 执行服务 → 关中断
→ 恢复现场+屏蔽字 → 开中断 → 返回

DMA 传送过程
plaintext
【1 预处理】
CPU 设置：
- 主存地址 → AR
- I/O 设备地址 → DAR
- 数据个数 → WC
启动 I/O

【2 数据传送】
DMA 接管总线
自动传输整块数据
CPU 继续执行程序

【3 后处理】
传输完成 → 发中断
CPU 执行中断服务程序
DMA 特点
主存 ↔ I/O 直接通路
不经过 CPU
整块传输、效率高
适合高速外设（磁盘）
总线占用方式：
停止 CPU 访存
周期挪用（最常用）
交替访存
DMA 与中断的区别
表格
项目	中断	DMA
控制方式	程序控制	硬件控制
CPU 介入	多（现场切换）	极少（前后处理）
响应时机	指令周期结束	总线空闲即可
适用设备	中低速	高速
优先级	较低	更高
功能	可处理异常	只负责数据传输
